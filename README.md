# LLM Service

Минимальный устойчивый сервис на FastAPI с поддержкой LLM, кешированием, повторными попытками и резервным ответом.

## 📌 Описание

Это сервис чата, который:
- Принимает сообщения пользователя
- Проверяет кеш
- При отсутствии кеша — обращается к LLM через API
- Реализует retry-логику при ошибках
- Возвращает резервный ответ при полном отказе
- Логирует все действия в JSON-формате

Разработан для надёжности и простоты масштабирования.

## 🚀 Быстрый старт

### 1. Установите зависимости
```bash
pip install -r requirements.txt
```

### 2. Настройте окружение
Создайте файл `.env` в корне проекта:
```env
LLM_API_KEY=ваш_ключ
LLM_BASE_URL=https://api.llm7.io/v1/chat/completions
LLM_MODEL=openai
LLM_TIMEOUT=10
CACHE_TTL_SECONDS=600
```

### 3. Запустите сервер
```bash
uvicorn main:app --reload --host 127.0.0.1 --port 8080
```

Сервис будет доступен по адресу `http://localhost:8080`.

## 🧩 API

### `GET /`
Проверка состояния сервиса.

**Ответ:**
```json
{
  "status": "ok",
  "service": "LLM Service"
}
```

### `GET /health`
Проверка работоспособности сервиса (health check).

**Ответ:**
```json
{
  "status": "healthy"
}
```

### `POST /chat`
Обработка сообщения пользователя.

**Тело запроса:**
```json
{
  "message": "Привет! Как дела?"
}
```

**Поля:**
- `message`: строка от 1 до 1000 символов

**Успешный ответ:**
```json
{
  "answer": "Привет! У меня всё отлично, спасибо!",
  "source": "llm",
  "execution_time_sec": 1.234
}
```

**Из кеша:**
```json
{
  "answer": "Привет! У меня всё отлично, спасибо!",
  "source": "cache",
  "execution_time_sec": 0.002
}
```

**При ошибке:**
```json
{
  "detail": "Сервис временно недоступен"
}
```


## ✨ Особенности

- **Кеширование** — повторные запросы отвечаются мгновенно
- **Retry (3 попытки)** — экспоненциальная задержка: 1, 2, 4 сек (макс 10 сек)
- **Fallback** — при сбое LLM возвращается статический ответ
- **JSON-логирование** — структурированные логи в консоль и файл `logs/app.log`
- **Валидация** — Pydantic-схемы для входных данных
- **Настройки через .env** — гибкая конфигурация
- **Отдельный логгер** — не использует root, избегает конфликтов
- **Middleware** — логирует все HTTP-запросы и ошибки
- **Жизненный цикл** — логирует startup и shutdown

- Ключ API (`LLM_API_KEY`) не жёстко закодирован

- Валидация входных данных
- `.gitignore` защищает от утечки .env и бинарных файлов
